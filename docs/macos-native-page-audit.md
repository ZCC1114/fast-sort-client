# macOS 原生页面审计与重排计划

更新日期：2026-07-06

目标：保留现有业务功能、接口调用和交互入口，把 macOS 客户端从 Web 复刻视觉改成更符合 macOS 的原生信息架构。

## 全局问题

- 左侧导航仍是 Web 的窄图标栏，macOS 上识别成本高，长标题容易缩放。
- 顶部栏只有右侧操作，页面标题在内容区，缺少 macOS 常见 toolbar/titlebar 信息层级。
- 卡片阴影和米色背景偏 Web，多个页面有“网页 dashboard”感，不像 macOS 分组面板。
- 多个页面使用固定宽度列和固定 HStack，窗口缩小时容易横向溢出。
- 表格类页面没有统一的表头、滚动和空状态层级。
- 设置、个人中心等页面用下划线 tab，不如 macOS sidebar/list selection 清晰。

## 页面问题与重排方向

### 登录

- 问题：表单卡片像 Web 登录框；输入控件不是 macOS 表单栅格。
- 重排：保留短信/账号登录、协议、验证码、发送验证码；改成居中原生表单面板。

### 首页

- 问题：5 列统计固定布局，窄窗口下压力大；右侧栏固定 420px。
- 重排：统计卡改 adaptive grid；报表、直播间、黑名单、批次使用原生 grouped cards。

### 直播端

- 问题：左房间列表、右信息区是 Web 卡片堆叠；按钮过多时拥挤。
- 重排：保留房间管理、开播、打印模式、弹幕、设置、淘宝链接、小红书/快手预检；布局为左侧 room source list + 右侧 inspector/work area。

### 娱乐模式

- 问题：控制面板和输出设置并列固定宽；弹幕列表区域不够像事件流。
- 重排：左侧房间列表、右侧顶部状态与控制、底部事件流，输出开关放入 inspector section。

### 理货端

- 问题：批次为空时左栏被隐藏导致布局跳变；明细表固定列宽。
- 重排：左侧始终显示批次 source list；右侧用工具条 + 可横向滚动明细表，保留搜索、分页、拉黑、重置编号。

### 订单一键备注

- 问题：平台、批次、字段开关混在左/右两个 Web 卡片里。
- 重排：左侧批次列表；右侧顶部字段选择，下面预览表和执行按钮，保留生成 JSON、打开后台。

### 黑名单

- 问题：筛选、范围 tab、列表、详情割裂；详情指标占空间。
- 重排：顶部 filter toolbar；左侧黑名单列表；右侧详情 inspector，保留全局/我的范围、等级、平台筛选。

### 充值记录

- 问题：表格固定列宽且无横向滚动容器。
- 重排：顶部状态 segmented control；表格放入原生滚动容器。

### 设置

- 问题：设置 tabs 是按钮列表，视觉和 macOS 设置侧栏不一致；内容区只是只读列表。
- 重排：左侧 settings sidebar；右侧 grouped form/list，保留直播间、模板、弹幕、映射、理货、黑名单设置入口和打印测试跳转。

### 个人中心

- 问题：顶部横向 tab 不适合多项账号设置；表单宽度固定。
- 重排：左侧账号设置 sidebar；右侧 Form 风格分组，保留昵称、密码、手机号、注销流程。

### 支付

- 问题：权益和套餐两列固定，视觉营销化。
- 重排：左侧权益说明，右侧套餐选择 inspector，保留 PC 支付打开临时 HTML。

### 打印测试

- 问题：打印机选择和指令内容固定 HStack，日志层级弱。
- 重排：左侧打印机/参数 inspector；右侧指令编辑器和日志，保留系统打印机枚举、预设指令、发送。
