# 迅拣 Windows/macOS 原生客户端开发计划

更新时间：2026-07-05  
参考基线：`docs/client-rebuild-context.md` 中记录的后端、Web、App 最新提交  
目标：按 Web 端 UI 风格和现有功能完整复刻，分别实现 Windows 原生客户端和 macOS 原生客户端，并在复刻稳定后逐项优化 Web/App 当前不好用的地方。

## 技术路线

### 平台选择

| 平台 | 技术栈 | 说明 |
| --- | --- | --- |
| Windows | .NET 8 + WPF + MVVM | 真正 Windows 原生桌面客户端；适合打印机、串口/USB、Windows 语音和安装包能力。当前机器可写代码但无法完整编译 WPF，最终验证需要 Windows + Windows SDK。 |
| macOS | Swift 6 + SwiftUI + AppKit 按需桥接 | 真正 macOS 原生桌面客户端；当前机器具备 Xcode，可本机编译验证。打印、语音、店铺自动化可用 AppKit/IOKit/NSSpeechSynthesizer 和独立自动化流程补齐。 |
| 共享部分 | 文档、接口契约、业务规则、设计 Token | 不共享 UI 代码。两端各自原生实现，但共享 API 路径、数据模型定义、打印协议说明和页面验收清单。 |

### 路线修正

- 不用 Electron/Tauri 作为主客户端，因为用户要求 Windows 和 Mac 分别写原生客户端。
- macOS 测试包必须采用全原生运行路径：登录、首页、直播端、娱乐模式、理货端、订单备注、黑名单、充值记录、设置、个人中心、支付、打印测试全部进入 SwiftUI/AppKit 原生页面。
- Web 项目只作为截图、接口链路和交互验收参照，不作为客户端页面回退方案。
- 订单备注、打印、语音和本地安全存储都需要客户端级能力；页面本身不能停留在 WebView 包壳形态。

## Web UI 风格复刻基准

- 布局：左侧窄导航、顶部用户/VIP/手册栏、主内容区卡片式信息密度。
- 色彩：暖白背景 `#fffaf3`、白色卡片、浅米边框、橙色主色 `#f18b00`、低饱和灰色辅助文本。
- 组件：圆角卡片、分段按钮、状态胶囊、表格分页、弹窗、全局成功/失败 Toast、加载遮罩。
- 字体：中文优先保证清晰；保留 Web 的 Sora/Plus Jakarta Sans 视觉倾向，原生端使用系统字体加字重/间距复刻。
- 交互：保留 Web 的页面结构和操作顺序；优化只在功能复刻验收通过后进入第二轮。

全量复刻验收口径：

- 左侧 8 个主菜单必须全部可点击、可进入真实页面，不能只有占位说明。
- 每个主菜单页面必须覆盖 Web 里的列表、搜索/筛选、分页、弹窗、确认、空态、加载态、错误态、成功提示。
- Web 顶部和底部入口虽然不在主菜单中，也按同等页面范围实现：个人中心、支付页、手册、打印测试、退出登录。
- 设置页下的 6 个标签和模板编辑器按独立页面级复杂度验收，不合并成一个简单表单。
- 直播端、娱乐模式、打印测试、模板测试打印必须接入原生打印能力；订单备注必须用原生自动化替代浏览器扩展。

## 页面功能清单

### 1. 登录页

- 顶部 Logo、左右插画、底部备案和版权信息。
- 短信登录和账号密码登录两个标签页。
- 手机号输入，自动去掉开头 `86`。
- 发送验证码：调用 `/app/user/generateCaptcha`，60 秒倒计时。
- 四位验证码输入框：自动跳转、退格回退、粘贴拆分。
- 账号密码登录：手机号 + 密码。
- 服务协议/隐私协议勾选校验和外链打开。
- 登录成功后保存 Token，调用 `/app/user/getProfile` 缓存用户信息，进入首页。
- 登录失败、未填手机号、未勾选协议、验证码/密码缺失提示。

### 2. 全局框架

- 左侧导航：首页、直播端、娱乐模式、理货端、订单一键备注、黑名单、充值记录、设置。
- 侧边栏底部：个人中心入口、退出登录按钮。
- 顶部栏：手册、升级 VIP、用户昵称、VIP/试用剩余天数、非 VIP 状态。
- VIP 临期提醒：只在首页触发，7 天/3 天提示跳转充值。
- 全局 API 成功/失败弹窗。
- 登录过期自动清理会话并回登录页。

### 3. 首页 Dashboard

- 统计卡片：打印标签数、扣费用户数、直播间数、标签模板数、黑名单数。
- 统计卡片点击跳转对应页面。
- 打印报表折线图：7 天、30 天、开始/结束日期筛选。
- 最新理货批次：按抖音、淘宝、小红书、微信、快手各取最新当前或历史批次。
- 未完成批次支持“重置编号/完成批次”确认操作。
- 直播间卡片：显示头像、名称、房间号、平台，点击进入直播端并选中房间。
- 黑名单摘要：我的黑名单/全局黑名单切换，全局需 VIP；分页、等级、平台、类型、创建时间。

### 4. 直播端

- 左侧直播间列表：平台标签、搜索、房间计数、头像、名称、房间号、播放/停止按钮。
- 添加直播间：
  - 抖音：输入房间号。
  - 淘宝：输入直播间名称，开播前弹出分享链接输入。
  - 小红书：保持当前 Web 行为，提示从 App/登录缓存获取。
  - 微信视频号：扫码登录、轮询扫码状态、检查直播间、保存会话。
  - 快手：输入房间号和 Cookie。
- 开播前校验：用户信息、VIP、直播间状态、房间打印模板是否配置。
- 平台接入：抖音 `/ws/{roomNumber}`、淘宝先 `/api/live/check_and_start` 后 `/tb-ws-room/{room_id}`、小红书 `/xhs-ws/{roomNumber}`、微信 `/wx-ws`、快手 `/ks-ws` 或后端返回 wsPath。
- 弹幕区：连接状态、弹幕状态、自动滚动、最多保留最近 50 条。
- 弹幕行状态：普通、等待打印、已打印、打印失败、黑名单拦截。
- 手动打印：点击弹幕，解析数字或映射值，调用 `/app/fsLiveTag/printLiveTagV2`，发送打印 Hex。
- 自动打印：
  - 自动/手动模式切换。
  - 粉丝团过滤、唯一模式、竞价模式、九宫格模式。
  - 频率控制：间隔秒数、每轮打印数量。
  - 重复用户过滤：按房间保存秒数。
  - 黑名单过滤：我的黑名单必拦截；全局黑名单按用户配置等级拦截。
  - 自动队列：断打印机暂停，重连恢复，状态诊断。
- 房间打印配置读取：标签宽高、指令类型、弹幕模板规则、弹幕映射规则。
- 悬浮设置：字号、自动打印间隔/数量、重复过滤、竞价等待秒数、九宫格窗口秒数。

### 5. 娱乐模式

- 仅展示有可用抖音 Cookie 的直播间。
- 房间列表：搜索、头像、名称、房间号、进入/退出。
- 当前房间信息：平台、房间号、连接状态、礼物明细摘要。
- 互动输出配置：进房、送礼、点赞分别控制打印和播报。
- 打印机面板：进入娱乐模式时切到 ESC/POS，退出后恢复原设置。
- 操作按钮：清空弹幕区域、配置模板、测试播报、测试声音、测试打印。
- 消息区：
  - 过滤：全部、弹幕、进房、礼物、打赏、榜单、点赞。
  - 支持进房、点赞、礼物、粉丝团、房间、榜单、PK、系统消息等归一化显示。
  - 点赞漂浮效果、进房提示、礼物动效。
- 礼物统计：去重、合并连送、弹窗展示全部礼物明细。
- 打印：
  - 进房/点赞文本批量合并打印。
  - 礼物打印包含头像、礼物图、数量，生成 ESC/POS 位图。
  - 图片热敏化处理、抖动、裁边、队列打印。
- 语音：
  - 礼物播报合并，同用户连续播报用“又送了”。
  - 优先本地语音代理，客户端原生改为 Windows/macOS 本地语音。
  - 支持按类型清空语音队列。
- 模板配置抽屉：进房提示、点赞提示、送礼提示；支持 token 和自定义文本拖拽排序。

### 6. 理货端

- 当前批次/历史批次切换。
- 平台切换：抖音、淘宝、小红书、微信、快手。
- 批次列表：当前未完成批次，历史分页。
- 批次行展示名称、创建时间，选中后加载标签明细。
- 明细搜索：昵称/编号。
- 明细表格：昵称、编号、总金额、数量、创建时间、更新时间、操作。
- 加入黑名单弹窗：
  - 类型：恶意/跑单。
  - 跑单金额必填且不能超过总金额。
  - 备注最多 1000 字。
  - 调用 `/app/fsBlack/addFsBlack`，成功后行状态变黑名单。
- 完成批次/重置编号：读取默认设置，支持勾选是否重置编号。

### 7. 订单一键备注

- 平台：抖音、小红书。
- 当前批次/历史批次切换，历史分页。
- 批次行展示名称和创建时间。
- 备注字段选择：昵称、编号、货架/序号、数量、金额；至少保留一个字段。
- 标签明细预览：搜索、分页、刷新；按编号升序。
- 备注生成规则：
  - 编号前缀 `#`，货架/序号前缀 `@`，数量前缀 `x`，金额前缀 `¥`。
  - 按字段顺序拼接备注。
  - 按平台选择 buyerId 字段：抖音优先 `remarkBuyerId/shortId/orderNameId`，小红书优先 `remarkBuyerId/danmuUserId/orderNameId/shortId`。
- 执行状态：扩展/桥接状态、已连接店铺、安装/连接指引。
- 原生客户端改造点：不能依赖 Chrome 扩展 `window.postMessage`，需要做原生店铺自动化或独立自动化进程。
- 执行批次：
  - 全量分页读取标签明细。
  - 构造 remarkMap。
  - 展示进度、成功/跳过/失败统计、失败原因。

### 8. 黑名单

- 搜索、等级筛选、平台筛选。
- 我的黑名单/全局黑名单切换，全局需 VIP。
- 列表：名称、平台、黑名单类型、创建时间、等级。
- 我的黑名单支持删除，带确认弹窗。
- 详情区：等级、跑单次数、恶意行为数、总跑单金额。
- 详情列表：订单名、黑名单类型、跑单金额、备注、创建人、更新时间。
- 首次加载自动选中第一条。

### 9. 设置页

#### 9.1 直播间设置

- 房间列表：平台标签、搜索、删除房间。
- 基础信息：首单邮费步进器。
- 标签模板：横向模板卡片、预览、单选；无模板时跳转新增模板。
- 弹幕模板：多选。
- 弹幕映射：单选“无”或某个映射规则。
- 打印速度：快速、稳定、普通。
- 保存：调用 `/app/fsUserRoom/updateFsUserRoomPostage`。

#### 9.2 标签模板

- 模板列表横向滚动、预览、选中、删除、新增。
- 内嵌模板编辑器。
- 保存后刷新列表并保持选中。

#### 9.3 弹幕模板

- 左侧弹幕模板列表、新增、删除。
- 中间规则构建区域：按元素顺序排列，可删除。
- 元素选项从 `/common/enum/danmuElement` 读取。
- 数字元素：随机、多值、范围；禁止连续数字元素。
- 文本/符号元素：支持自定义内容或长度限制。
- 右侧预览：随机生成命中/未命中弹幕。
- 保存新增/更新，删除确认。

#### 9.4 弹幕映射

- 规则搜索。
- 映射规则卡片：名称、映射项、编辑、复制、删除。
- 新增/编辑弹窗：组名、映射行，支持增删行。
- 校验：名称必填、映射文本不重复、至少一条有效映射。

#### 9.5 理货端设置

- 货架编号区间卡片：编号、开始、结束。
- 添加/删除货架，至少保留一个。
- 校验：编号必填且不重复、区间必填、开始不能大于结束、区间不能重叠。
- 示例图片预览。
- 保存货架范围和是否重置编号。

#### 9.6 黑名单设置

- 是否打印过滤黑名单。
- 全局黑名单等级阈值。
- 黑名单规则提示。
- 保存用户黑名单设置。

### 10. 标签模板编辑器

- 组件库：从 `/common/enum/tagElement` 读取，图标、描述、备注气泡。
- 画布：按标签宽高和 DPI 换算像素，支持缩放 50/75/100/125/150%，网格线开关。
- 拖拽：从组件库拖到画布，画布内拖动，限制不越界。
- 元素编辑浮层：标题、测试内容、字号、加粗、镜像、删除。
- 标签规格：名称、宽高、DPI 203/200/300、指令 TSPL/CPCL、方向 0/180。
- 时间元素每秒刷新。
- 模板布局保存为 `templateLayout` JSON：宽高、dpi、instructionType、rotate、elements。
- 测试打印：调用 `/app/fsLiveTag/textPrintLiveTagV2` 生成 Hex 后发送打印机。
- 校验：名称、宽、高必填；宽最大 90mm，高最大 150mm。

### 12. 充值记录

- 分页和每页数量切换。

### 13. 个人资料

- 标签：关于、修改昵称、修改密码、修改手机号、注销账号。
- 关于：昵称、手机号、VIP 到期时间、邀请码、检查更新、备案、联系邮箱、客服二维码预览、版权。
- 修改昵称：必填，未变化不提交。
- 修改密码：确认密码、一致性、长度 6-20、不能含中文；成功后退出登录。
- 修改手机号：新手机号、验证码、60 秒倒计时；成功后退出登录。
- 注销账号：当前手机号验证码、二次确认；成功后退出登录。

## API 迁移清单

第一阶段先以 Web `src/api/services.js` 为准，生成原生端服务类：

- `authApi`：登录、验证码、Profile、资料更新、注销、退出。
- `statsApi`：用户统计、标签趋势。
- `roomsApi`：直播间查询、状态、增删改、平台缓存、邮费/模板配置。
- `liveApi`：直播开始/结束、弹幕、标签查询、打印、测试打印。
- `sortApi`：批次、完成批次、理货设置。
- `blackApi`：黑名单、详情、用户黑名单过滤设置。
- `templateApi`：标签模板、弹幕模板、弹幕映射、枚举。
- `vipApi`：VIP 套餐、订单、订单状态、邀请码。
- `paymentApi`：支付宝 PC/Web/H5/Apple 下单。

## 验收规则

- 页面布局、信息密度、色彩和交互顺序与 Web 一致。
- 每个页面至少覆盖空数据、加载中、成功、失败、登录过期状态。
- 所有后端接口路径和请求参数与 Web/App 对齐。
- 打印相关页面必须用真实或模拟打印设备验证字节输出。
- 直播端必须验证断线重连、自动队列、黑名单拦截、打印机断开恢复。
- 不把 Web 的浏览器限制原样带进客户端；但第一版行为结果必须一致。
